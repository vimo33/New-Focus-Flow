# Screen Interactions + UX Upgrades (per screen)

This is the “what happens on each screen” contract.

---

## A) Think

### 01 — Think: Strategy
**Purpose:** portfolio-level summary + top decisions.
**Primary interactions:**
- Filter/sort ventures by signal, stage, urgency.
- Click venture card → Project Details.
- “Run Council” CTA → opens Council Composer (defaults to idea_validation).
**UX upgrades:**
- Add “Top 3 Decisions Pending” module (one-liners + confidence chips).
- Add consistent evidence drawer for any recommendation.

### 02 — Think: Ventures (Portfolio list)
**Purpose:** list/board of clusters + variants.
**Primary interactions:**
- Create Cluster / Add Variant (wizard)
- Archive variant (non-destructive)
- Mark as “Active / Parked”
**UX upgrades:**
- Cluster grouping with collapsible variants.
- Quick inline “3-line summary + expand” for each project.

### 03 — Think: Finance
**Purpose:** cash/runway/burn, allocations, simple forecasts.
**Primary interactions:**
- Edit baseline numbers (with approval if it changes global constraints).
- Run forecast (safe) → generates report artifact.
**UX upgrades:**
- Use Observatory design language: tiles + breakdown + suggestions card w/ confidence.

### 04 — Think: Comms
**Purpose:** drafts/outreach queue; always approval-gated.
**Primary interactions:**
- Draft message → approval card
- Approve sends (later), now just queues.
**UX upgrades:**
- Adopt “Pending Approvals” list style from Morning Briefing.

---

## B) Validate

### 05 — Validate: Experiment Stack
**Purpose:** list experiments + “decision required” queue.
**Primary interactions:**
- Create experiment (variant testing)
- View experiment detail (metrics, cohorts, evidence)
- Decision Gate: Scale / Variant / Park / Kill (approval if impactful)
**UX upgrades:**
- Add “Decision Required” section at top with countdown + recommended action.
- For each decision: show Why/Source/Confidence.

### 06 — Validate: Variant Testing
**Purpose:** configure variants + rollout rules.
**Primary interactions:**
- Set variant parameters
- Define guardrails + rollback
- Launch test (approval if it triggers spend or prod deployment)
**UX upgrades:**
- Inline “Rollback plan” preview panel (always visible).

### 07 — Validate: Data Sources
**Purpose:** connectors + freshness + health.
**Primary interactions:**
- Connect / refresh / disable source (approval if external write).
- Inspect ingest logs.
**UX upgrades:**
- Status chips for freshness + last sync + failures.
- “Explain impact” button: what breaks if this source is down.

## C) Build

### 08 — Build: Build (orchestrator)
**Purpose:** turn validated decisions into PRD/spec/code tasks.
**Primary interactions:**
- Build plan preview (steps + costs + risk)
- Approve build kickoff (approval)
**UX upgrades:**
- Use Autonomous Builder log layout: feed + execution preview panel.

### 09 — Build: Control
**Purpose:** policies, gates, setup mode, budgets.
**Primary interactions:**
- Toggle Setup Mode (typed confirm)
- Adjust approval policy (approval)
**UX upgrades:**
- Make Setup Mode prominent with “allowed actions” summary.

### 10 — Build: Agents
**Purpose:** agents roster + permissions + tools.
**Primary interactions:**
- Enable/disable agent
- Assign to project/variant scope
- View agent memory scope
**UX upgrades:**
- Add “tool permissions” view: what each agent can execute.

### 11 — Build: Data
**Purpose:** internal data, logs, artifacts.
**Primary interactions:**
- Search artifacts
- Pin to evidence drawer
**UX upgrades:**
- Add “artifact provenance” (generated by which run, when, inputs).

### 12 — Build: Tune
**Purpose:** model settings, voice persona, prompt presets.
**Primary interactions:**
- Choose archetype voice (Strategist/Co‑founder/Critic)
- Preview response style
**UX upgrades:**
- Reuse Archetype Selection screen design from old UI.

## F) Council Verdict screen (priority UX fix)
**Issue:** long project overview pushes other elements down.
**Fix design:**
- Top section shows:
  - Title, stage, status chips
  - **Project Summary** (3-line clamp) + buttons:
    - “Expand” (inline accordion)
    - “Pop out” (right drawer / modal with formatted markdown)
- Council panel stays above fold:
  - composite score ring
  - dimension chips (Architect/Strategist/Analyst)
  - risks/open questions in a right column
  - recommended actions as checklist with “Create Tasks” toggles
**Formatting:**
- render overview as markdown with headings + bullets
- provide “copy summary” button for reuse

---

## G) Project Details screen (priority UX fix)
**Issue:** overview text dominates layout and hides actionable blocks.
**Fix layout:**
- Left: Overview card w/ 3-line clamp + expand/drawer
- Center: Next Decision + Next Actions (top)
- Right: Evidence / Council / Experiments (sticky)
**Primary interactions:**
- Run council (choose decision type)
- Create experiment stack draft
- Approve actions (gated)
